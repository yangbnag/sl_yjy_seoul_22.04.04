1조의 디데이 계산기 발표를 맡은 양재영 입니다.

저희는 프로젝트 개요, 프로젝트 소개, 개선방향 Q&A 순으로 발표를 준비했습니다.

1. 프로젝트 개요
1-1. 주제 선정 배경

 - 현대를 살아가는 사람들에게 하루는 매우 빠르게 지나갑니다. 
   그러면서 중요한 기념일들을 잊게되고 상대방에게 서운함을 주거나 
   싸움을 발생 시키기도 하며 
   시험 날짜가 코앞으로 다가오는 것을 인지 하지 못하고
   시험준비 시기를 놓쳐 낙방하는 일들이 발생되곤 합니다.

   이런 일들을 예방하고자 현재 시간과 달력 그리고 다가오는 날을 한눈에 볼수 있도록 D-DAY 달력을 만들보았습니다.

1-2. 목표 및 기대효과

 - D-DAY 달력을 통해 사용자는 시간과 날짜 그리고 기념일들을 한눈에 확인할수 있으며 
   철저한 시간관리를 통해 자신을 더욱 발전시킬수 있게 될 것입니다.


1-3. 팀원 소개 및 역할
   저희 팀은 고현정, 손성빈, 양재영 이렇게 3명으로 구성되어 있고,

 - 고현정씨는 HTML기초 구성제작과 전반적인 CSS를 담당하였고, JS로 디데이 입력 부분의 기능을 제작 하였습니다.
 - 손성빈씨는 js로 아날로그 시계와 디지털 시계 구현했고, 디데이 출력 기능과 피피티 제작을 담당하였습니다. 
 - 저는 JS로 달력과 오늘의 날짜,요일을 구현했고, 하단 입력창과 출력창의 기능구현시 문제점들을 보완하였습니다.

1-4. 기초 설계
  프로젝트를 진행할때 만들고자 했던 레이아웃 입니다.

1-5. 프로젝트 수행일정
=> 요일별 프로젝트 수행단계를 기술. 

팀프로젝트 진행 
 - 1일차 어떤 프로그램을 개발할 것인지 팀회의 진행.
 - 2일차 역할을 크게  (HTML과CSS구성 및 제작, 시계, 달력)  3가지로 분담하여 개발을 진행하였습니다.
                                                                               /// 현정 : HTML과 CSS 구성 및 제작 / 성빈 : 시계  / 재영 : 달력
 - 3일차에는 각자 만든 분야 완성 및 디데이 기능을 구현하였고,
 - 4일차부터 테스트를 통해 입출력 기능을 보완해 나가면서 마무리 CSS와 PPT를 제작 하였습니다.

/( 새로 생성된 객체 id 부여, 입력시 입력창 text값 초기화 / 출력창 삭제시 객체 삭제 시키기. )
 
    현정 : 오른쪽 하단 입력창 기능 구현
    성빈 : 왼쪽 하단 출력창 기능 구현
    재영 : 달력 투데이 출력 기능 추가 및 시계와 달력 메인 HTML에 넣는 작업 


1-6. 개발환경

 -저희 팀은 HTML CSS JS를 사용하여 디데이 계산기를 개발 하였습니다.

2. 프로젝트 절차

2-1. 핵심 기능 및 기술 소개

  저희팀이 만든 디데이 달력 입니다.

 - 웹에 현재 시간과  오늘 날짜 그리고 입력한 날짜가 몇일 남았는지 알수 있게 디데이 출력창을 만들어
   사용자가 웹에 접속했을 때 한눈에 볼수 있도록 하였습니다.
 
 - 시계는 아날로그와 디지털로 두가지 형식으로 출력 하도록 하였고
   달력은 이전달이나 다음달로 넘겼을 때 시작 일자와 요일이 어긋나지 않도록 하며
   토요일과 일요일은 글씨에 색깔을 넣어서 평일과 구분 하였습니다.

  - 디데이 입력시 디데이명의 입력값이 15자 이상 넘어가지 않도록 설정 하였습니다.

4-1. 프로젝트 개선사항
=> 향후 업데이트에서 추가할 기능이나 추가하고 싶은 기능들을 기술.

 - 디데이를 입력하면 달력에 동그라미가 그려지고 그 밑에 디데이내용이 출력되게 하는 기능.

 - 달력을 눌러서 디데이를 입력 할 수 있도록 하는 기능.

 - 이전 달 다음 달 넘기는 창을 년단위로도 쉽게 넘길수 있도록 하는 기능.

 - 디데이 시작 날짜와 종료 날짜를 입력하지 않으면 입력되지 않도록 하는 기능.

 - 입력한 디데이 수정기능 및 남은 시간 출력 기능 

4-2. 피드백

- 개발 아이디어를 선정 하는 것 부터 깃허브를 통한 데이터 공유 그리고 기능 추가할 때 js 사용 미숙과 버그로 인해 계속해서 막히는 것 까지
  프로그램을 만드는 첫 팀프로젝트 여서 어려운 부분이 많았던 것 같습니다.

  수업시간에 배웠던 내용들이었는데도 기억이 안나고 버그가 발생하면 선생님께 도움을 요청하고 또 이렇게 해결해 나가면서 
  아직 갈길이 멀다고 느꼈습니다.

  이번 프로젝트를 하면서 제일 힘들다고 느꼈던건 제가 아닌 다른 사람이 만든 코드를 해석하는 것이었습니다.
  달력,시계 디데이 입력, 출력의 기능들은 처음 짜보거나 한번 배웠던 코드들이라서 
  인터넷에 있는 코드나 수업시간에 했던 코드를 참고 하면서 만들었는데 생각했던 대로 작동이 안되거나
  버그가 발생하면 작성자의 의도를 정확히 이해하지 못한 상태이기 때문에 문제들을 해결하는데 많이 애를 먹었습니다.

  아쉬웠던 점은 추가 시키고 싶은 기능은 많은데 한계에 부딪쳐 더 하지 못한점이 많이 아쉬웠습니다.
  다음 프로젝트때는 이런 아쉬움이 남지 않도록 수업시간에 배운 내용들을 잘 활용할수 있도록 많이 연습해야겠다고 느꼈습니다.
